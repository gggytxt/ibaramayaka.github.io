BFC

https://www.cnblogs.com/qs-cnblogs/p/12349887.html
https://blog.csdn.net/sinat_36422236/article/details/88763187hex

直译为”块级格式化上下文”。

  • 一个独立的布局环境，只有Block-level box参与，BFC规定了内部的Block-level Box如何布局，BFC内部的元素布局与外部互不影响。
  • 是Web页面的可视化CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。
  • 可以通过一些条件触发BFC，从而实现布局上的需求或解决一些问题；
  • 可以将BFC想象成一个工具，无需探究其定义，只要着重理解其功能（特性）即可。

需要先了解Box、Formatting Context

Box：css布局的基本单位

  • Box 是 CSS 布局的对象和基本单位，一个页面可以看成是由很多个 Box 组成的。
  • 元素的类型和 display 属性，决定了这个 Box 的类型。不同类型的 Box，会参与不同的 Formatting Context（一个决定如何渲染文档的容器），因此Box内的元素会以不同的方式渲染。有哪些box：
    1.block-level box:display 属性为 block, list-item, table 的元素，会生成 block-level box。并且参与 block fomatting context；
    2.inline-level box:display 属性为 inline, inline-block, inline-table 的元素，会生成 inline-level box。并且参与 inline formatting context；
    3.run-in box: css3 中才有， 这儿先不讲了。

Formatting Context

Formatting context 是 W3C CSS2.1 规范中的一个概念。

  • 它是页面中的一块渲染区域，并且有一套渲染规则，它决定了其子元素将如何定位，以及和其他元素的关系和相互作用。
  • 最常见的 Formatting context
    1.Block fomatting context (简称BFC)
    2.Inline formatting context (简称IFC)。

形成BFC的条件

1.根元素（html）
2.float值非none
3.overflow值非visible（hidden，auto，scroll）
4.display值为inline-block、table-cell、table-caption、flex、inline-flex
5.position值不是static或者relative，为absolute、fixed

BFC的特性

1、Box垂直方向的距离由margin决定，属于同一个BFC的两个相邻容器的上下margin会重叠（重点）
2、计算BFC高度时浮动元素也参于计算（重点）
3、BFC的区域不会与浮动容器发生重叠（重点）
4、BFC内的容器在垂直方向依次排列
5、元素（块盒与行盒）的margin-left与其包含容器的border-left相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
6、BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。

IFC介绍

https://blog.csdn.net/u011472830/article/details/73010596
https://blog.csdn.net/weixin_38080573/article/details/79364754

IFC(Inline-Formatting-Contexts)即内联格式化上下文布局。
一般都是行内元素比如：input a img span 以及display 属性值为inline-block的元素。

行内元素的特点：

  • 水平方向根据direction依次布局。
  • 不会在元素前后换行。
  • 受white-space属性的影响。
  • margin/padding 在竖直方向无效，水平方向有效的。
  • white/height 对非替换行内元素无效，宽度由元素内容决定。（替换元素在下面解释）
  • 非替换行内元素的行框高由line-height决定而替换行内元素的行框高则是由height，padding，border，margin决定
  • 浮动或者绝对定位会转化为block
  • vertical-align属性生效

替换元素理解是元素本身没有实际的东西，最终显示出来的东西需要浏览器根据自身的某一个属性去判断。比如img属性，最终显示的东西由src这个属性值来决定的。再如input元素，最终展现出的东西事由type属性值来决定的。textarea,select、object等这些都是替换元素。

主要影响IFC行内布局的css：

  • font-size
  • line-height
  • height
  • vertical-aligin

IFC特性

1.框会从包含快的顶部开始一个接一个的排列。
2.放置这些框的时候，他们在水平方向上的外边距，边框和内边距有效。垂直方向上这些框可能会以不同的方式来对齐，比如顶部对齐、底部对齐、按照文本的基线对齐。一行上的框都包含进去的一个大框称为改行的行框。水平的padding,border,margin有效，竖直方向无效。不能指定框高。
3.行框的宽度由包含块和存在的浮动决定，高度至少足以包含他内部的所有框。它也许比包含在它内最高的盒子高(盒子对齐导致基线提高了)。
4.当一行上的行级总宽度小于行宽的时候，他们在行宽内水平方向上的排布由text-align决定。如果属性是’justify’，用户代理可能会拉伸空间和文字在inline boxs内。
5.当一个行内框的宽度超过了该行行宽，就会被分成几个框(文字换行)。如设置框不可分割（文字强制不给换行white-space设置为nowrap）那么该行内框会溢出该行行宽。
6.一般情况下行宽左边紧贴他的包含块的左边，同样他的右边也是紧贴其包含块的右边。 出现浮动时，浮动元素可能会插在包含块和行框之间。
一个IFC中行框高度通常是变化或不一定的，比如某一行的某个框是个很高的图片，而改行框中其他框只是文字。
7.计算行框内各个框的高度，对于非替换元素就是起line-height,而对于替换元素就是边界框的高度了。
